# 拼豆图案转换器 — 改进计划书

> 创建日期：2026-04-03
> 状态：规划中

---

## 一、核心认知

拼豆图的本质矛盾在于：**原图是为高分辨率设计的，而拼豆是一个极低分辨率的表达介质**。简单的"缩小 + 颜色映射"永远无法弥合这个鸿沟，真正好看的拼豆图需要"像素级的二次创作"。

因此，工具的核心定位应从 **"图像转换器"进化为"像素画辅助创作工具"**——自动生成只是起点，真正的价值在编辑体验。

### 理想工作流

```
原图 → 自动生成（粗稿）→ 参考原图进行二次绘画 → 完成
```

## 三、痛点分析

### A. 算法层面

| # | 痛点 | 说明 |
|---|------|------|
| A1 | 颜色映射质量 | 逐像素最近邻匹配无法表达区域整体色调；缺少误差扩散(Dithering)算法；色板色域有限导致某些颜色大面积色偏 |
| A2 | 缩放算法单一 | 区域平均降采样对像素画/卡通/线稿类图像会模糊边缘和线条；缺少针对不同图像类型的自适应缩放策略 |
| A3 | 尺寸选择盲目 | 用户不知道多大尺寸才能保留足够细节，只能反复试错 |

### B. 编辑体验

| # | 痛点 | 说明 |
|---|------|------|
| B1 | 手动修正效率低 | 缺少全局颜色替换、区域选择+批量替换；无法高效进行大面积修正 |
| B2 | 二次创作缺少参考 | 编辑时看不到原图，只能凭记忆修改；无法"对着原图画" |
| B3 | 大图导航困难 | 超过60×60时缩放平移频繁；缺少小地图/鹰眼视图 |

### C. 工作流层面

| # | 痛点 | 说明 |
|---|------|------|
| C1 | 色板与实际库存不匹配 | 用户手里不一定有色板所有颜色；缺少"子集选择"功能 |
| C2 | 材料清单不完善 | 缺少按包装规格换算、成本估算、可导出的采购清单 |
| C3 | 打印引导不足 | 缺少分板打印（按29×29拼豆板拆分）和1:1打印校准 |

## 四、改进计划

### Phase 1：全局颜色替换 ✅ 已完成

**目标**：解决"某个颜色整体偏了"的高频修正需求

**功能描述**：
- 选择一个源颜色 → 选择一个目标颜色 → 一键替换全部
- 可选"仅替换选区内"（待 Phase 3 选区功能实现后）
- 支持撤销

**已实现内容**：
- `REPLACE_COLOR` action 及 reducer 逻辑（遍历 grid 替换所有匹配颜色）
- `ColorReplacer` 组件：两步式交互（选源颜色 → 选目标颜色 → 替换全部）
- 优先展示图案中已使用的颜色，支持搜索全部色板颜色
- 支持替换为空（擦除）
- 完整支持撤销/重做
- 集成在侧边栏，生成图案后可用

**完成日期**：2026-04-08

**解决痛点**：B1

---

### Phase 2：原图参考层叠加（Reference Overlay）🔴

**目标**：让用户在编辑拼豆图时能同时看到原图，实现"对着原图画"

**功能描述**：
- 在 BeadGrid 上叠加一层半透明的原图
- 可调透明度（0%~100%）
- 快捷键切换显示/隐藏（如按住空格临时显示）
- 原图自动对齐到拼豆网格的位置和大小

**实现思路**：
- 在 BeadGrid 的 Canvas 上增加一个叠加层
- 将原图缩放到与拼豆网格相同的像素尺寸，居中对齐
- 通过 CSS opacity 或 Canvas globalAlpha 控制透明度

**预估工作量**：中（1~2天）

**解决痛点**：B2（核心体验质变）

---

### Phase 3：矩形选区工具（Rectangle Select）🔴

**目标**：从"逐点修补"升级为"区域创作"

**功能描述**：
- 新增选区工具，可框选矩形区域
- 选区内可批量填充颜色
- 选区内可批量颜色替换（结合 Phase 1）
- 选区可复制/粘贴/平移
- 选区可删除（设为空）

**实现思路**：
- 新增 `selection: { x: number, y: number, width: number, height: number } | null` 状态
- 选区工具的鼠标交互：mousedown 记录起点，mousemove 绘制选区框，mouseup 确认选区
- 选区内操作通过 SET_CELLS action 批量修改

**预估工作量**：中高（2~3天）

**解决痛点**：B1

---

### Phase 4：色板子集选择 + 限色生成 ✅ 已完成

**目标**：让用户只用手头有的颜色生成图案；减少近似色，让画面更干净

**功能描述**：
- 从色板中勾选"我有哪些颜色"，只用选中的颜色生成
- 提供"自动限色"：分析图像主色调，推荐 8/16/24/32 色子集
- 限色后重新生成

**已实现内容**：
- `PaletteSubsetSelector` 组件：限色生成开关 + 手动选色 + 自动推荐
- `auto-limit-colors.ts` 模块：k-means++ 聚类提取图像主色调，从色板中匹配最近 N 色
- `color-match.ts` 新增子集匹配器：`initSubsetMatcher` / `findNearestColorWithSubsetMatcher`，基于子集构建独立 k-d 树 + 量化缓存
- `image-processor.ts` 的 `processImage` 支持可选 `subsetIndices` 参数
- 自动推荐支持 8/16/24/32 色预设，自动包含黑白色
- 手动选色支持搜索、按分类分组、全选/全不选
- 切换色板时自动重置子集选择
- 集成在侧边栏，生成图案前可配置

**完成日期**：2026-04-08

**解决痛点**：C1、A1

---

### Phase 5：轮廓强化 / 边缘锐化 🟠

**目标**：自动生成时保留更清晰的轮廓线，更接近像素画美学

**功能描述**：
- 生成时提供"轮廓强化"选项
- 先对原图做边缘检测（Sobel/Canny），在缩放时优先保留边缘像素的颜色
- 或提供"自动描边"后处理：检测色块边界，用深色勾勒轮廓

**实现思路**：
- 在 image-processor 中增加边缘检测预处理步骤
- 边缘像素使用最近邻插值（保留锐利边缘），非边缘像素使用区域平均（保留色调）
- 后处理描边：扫描 grid，在相邻颜色差异大的位置插入深色轮廓

**预估工作量**：中（2~3天）

**解决痛点**：A2

---

### Phase 6：多级尺寸预览 🟠

**目标**：帮助用户选择合适的拼豆板尺寸，减少试错

**功能描述**：
- 上传图片后，一次性生成 3 个尺寸的缩略预览（如 30×30、50×50、70×70）
- 让用户直观对比后选择
- 标注推荐尺寸

**实现思路**：
- 在 BoardConfig 旁增加"尺寸预览"按钮
- 后台并行生成 3 个尺寸的低分辨率预览（可用 Web Worker 加速）
- 以缩略图形式展示对比

**预估工作量**：中（1~2天）

**解决痛点**：A3

---

### Phase 7：误差扩散 / Dithering 选项 🟡

**目标**：用有限色板在视觉上"混合"出更多颜色层次

**功能描述**：
- 生成时提供 Dithering 开关和强度调节
- 支持 Floyd-Steinberg 等经典误差扩散算法
- 需要在 dithering 强度和清晰度之间找平衡

**实现思路**：
- 在 image-processor 的颜色映射步骤中，加入误差扩散逻辑
- 每个像素匹配最近色后，将色差按权重分配给相邻未处理像素
- 提供强度系数（0~1）控制扩散比例

**预估工作量**：中（1~2天）

**解决痛点**：A1

---

### Phase 8：分板打印 + 拼接指引 🟡

**目标**：大图案按实际拼豆板尺寸拆分打印，带拼接指引

**功能描述**：
- 按标准拼豆板尺寸（如 29×29）自动拆分图案
- 每板单独一页 PDF，带板号标注
- 生成拼接总览图，标注各板位置关系
- 支持 1:1 打印校准

**实现思路**：
- 在 export-pdf 中增加分板逻辑
- 计算需要多少块板（ceil(width/29) × ceil(height/29)）
- 每块板生成独立页面，标注行列编号

**预估工作量**：中（2~3天）

**解决痛点**：C3

---

### Phase 9：采购清单导出 🟡

**目标**：帮助用户计算需要购买的材料

**功能描述**：

## 六、设计原则

1. **自动生成是起点，不是终点** — 所有功能设计都应服务于"辅助二次创作"
2. **编辑效率优先** — 减少重复操作，支持批量修改
3. **所见即所得** — 编辑时能实时看到效果，减少认知负担
4. **可撤销** — 所有操作都支持撤销/重做，鼓励用户大胆尝试
5. **渐进式复杂度** — 基础功能开箱即用，高级功能按需展开
